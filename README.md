# CS 250 Portfolio Artifact – Sprint Review and Retrospective

This repository contains my Sprint Review and Retrospective from the SNHU Travel project. This artifact demonstrates my understanding of Agile and Scrum practices, including iterative development, user stories, team roles, and retrospective analysis.

## Reflection

### Interpreting User Needs and User Stories
User stories help translate user needs into actionable development tasks by framing requirements from the user’s perspective. This ensures development focuses on delivering meaningful functionality that aligns with user expectations.

### My Development Approach and Agile Practices
I approach program development using an iterative process, building functionality incrementally and refining it through feedback. Agile practices such as sprint planning, backlog prioritization, and retrospectives help maintain progress and improve software quality.

### Being a Good Team Member
Being a good team member means communicating clearly, collaborating effectively, and supporting shared project goals. Agile development requires transparency, accountability, and adaptability to ensure successful project outcomes.
